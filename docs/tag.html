在GitHub Pages上发布博客

简单几步，免费托管你的个人博客

新建博客文章步骤

1 准备文章内容

使用Markdown格式编写你的博客文章，这是GitHub Pages最常用的格式。确保文章包含标题、内容和适当的格式。

# 文章标题

这是文章内容，支持**加粗**、*斜体*等Markdown格式。

## 二级标题
- 列表项1
- 列表项2

2 命名文章文件

将Markdown文件命名为`YYYY-MM-DD-文章标题.md`格式，例如`2023-09-15-javascript-async.md`，并保存在`_posts`目录下。

3 添加文章头部信息

在文章开头添加YAML头部信息，指定标题、日期、分类等元数据：

---
layout: post
title: "JavaScript异步编程全解析"
date: 2023-09-15 10:30:00
categories: JavaScript
tags: async promise
---

4 提交到GitHub仓库

将新文章提交到你的GitHub Pages仓库：

git add _posts/2023-09-15-javascript-async.md
git commit -m "添加JavaScript异步编程文章"
git push origin main

5 查看发布结果

提交后，GitHub Pages会自动构建并部署你的博客。几分钟后，访问`https://你的用户名.github.io`即可看到新发布的文章。

推荐工具

  • Jekyll - GitHub Pages默认支持的静态站点生成器
  • VS Code - 带有Markdown预览功能的代码编辑器
  • Prose - 用于编辑GitHub上Markdown文件的在线编辑器
  • GitHub Desktop - 简化Git操作的桌面客户端
